# ALLBLU 기능명세서 V4

## 0. 문서 목적
이 문서는 ALLBLU MVP 구현을 위한 기능명세를 정리한 기준 문서다.  
원문 요구사항을 구현자가 빠르게 범위 판단할 수 있도록 `Must`, `Nice`, 공용 컴포넌트, 화면 단위, 데이터/상태 정책으로 재구성했다.

## 1. 우선순위 기준
| 태그 | 의미 |
| --- | --- |
| Must | MVP 필수. 없으면 서비스 핵심 흐름이 성립하지 않는 기능 |
| Nice | 스코프 조절 대상. 스켈레톤만 구현하거나 후순위로 미룰 수 있는 기능 |

### 2-1. 화면 관찰 메모
공통 레이아웃:
- 좌측 상단 ALLBLU 로고, GNB 메뉴 `홈`, `웹툰`, `애니`, `유저추천`.
- 우측 검색 바와 계정 아이콘.
- 전체 화면은 흰 배경 중심, 카드/섹션은 얇은 border와 둥근 모서리 사용.
- 하단 푸터는 검정 배경, 출처/API 고지와 팀 고지를 흰 텍스트로 노출.

메인 홈:
- 상단 히어로는 짙은 남색 그라데이션 배너.
- `인기작 순위 TOP 20`은 6개 카드 단위의 가로 캐러셀.
- `인기평가`는 3개 카드 그리드.
- 최근 추가 추천 애니/웹툰은 각각 가로 캐러셀 형태.

로그인/회원가입:
- 중앙 정렬된 단일 폼.
- ALLBLU 로고를 폼 상단에 배치.
- 입력 필드는 밑줄형 스타일.
- 주요 CTA는 짙은 남색 full 버튼.
- 로그인 실패, 중복 ID, 비밀번호 길이 오류 등은 필드 근처 빨간 문구로 표시.
- 회원가입 완료 화면은 간단한 안내 문구와 로그인 버튼만 노출.

웹툰/애니 화면:
- 인기작 순위, 완결/시작/최근 추가 같은 카테고리별 추천 섹션은 가로 캐러셀.
- 전체 탭은 장르 필터와 30개 단위 그리드.
- 웹툰 연재중은 플랫폼별 섹션을 나눠 보여주는 구조.
- 완결/전체 그리드는 하단 도달 시 30개씩 추가 로드하는 무한 스크롤 구조.

썸네일 상태 드롭다운:
- 모든 작품 썸네일은 동일한 회색 포스터 비율과 우하단 원형 `+` 상태 버튼을 사용한다.
- `+` 클릭 시 드롭다운 표시: 볼 예정, 보는 중, 완료, 중단, 취소.
- 상태 선택 후 `+` 아이콘이 각 상태 아이콘으로 교체된다.
- 드롭다운은 썸네일, 마이페이지 애니/웹툰 보관함 모두 동일하게 적용한다.

비회원 잠금/무한 스크롤:
- 비회원도 대부분의 작품 목록과 썸네일을 볼 수 있다.
- 회원 전용 기능 클릭 시 로그인/회원가입 유도 영역 또는 토스트를 노출한다.
- 무한 스크롤 정책은 가로/세로 모두 동일한 조건으로 통일한다.
- 세로 그리드는 첫 로드 30개, 이후 30개씩 추가한다.
- 세로로 나열되는 추천 목록은 첫 로드 5개, 이후 5개씩 추가하는 변형을 허용한다.

작품 상세:
- 상세 페이지 내 `이 작품 좋아하시면 이것도 좋아하실지 몰라요!` 추천은 무한스크롤이 아니다.
- 상세 추천은 6개만 보이고, `전체보기` 클릭 시 유저추천 상세/올블픽 목록으로 이동한다.
- 회원은 `내 추천 작성하기`를 통해 올블픽 작성 화면으로 이동한다.
- 비회원은 평가 입력 placeholder를 로그인 안내로 바꾸고, 별점/작성창을 비활성화한다.
- 작품 순서 섹션은 별도 목록 형태로 제공한다.
- 평가 상세는 작성자 본인에게 `수정/삭제`, 타 유저에게 `공감/팔로우` 액션을 노출한다.

## 3. 전체 MVP 핵심 흐름
1. 사용자는 홈에서 애니/웹툰 추천과 검색 진입점을 확인한다.
2. 작품 썸네일을 클릭해 작품 상세 페이지로 이동한다.
3. 회원은 작품 상태를 `볼 예정`, `보는 중`, `완료`, `중단`으로 저장한다.
4. 비회원이 회원 전용 기능을 누르면 로그인/회원가입으로 유도된다.
5. 회원은 작품 평가를 작성하고, 마이페이지에서 보관함과 내가 쓴 글을 확인한다.
6. 회원은 올블픽에서 자신이 본 작품 기반으로 작품 간 추천을 등록하거나 동의한다.

## 4. 공용 컴포넌트

### 4-1. 작품 썸네일 카드 `CMP-THUMB-001`
적용 범위: 인기작 순위, 최근 추가 추천, 검색 결과, 카탈로그, 마이페이지 보관함 등 모든 작품 카드.

Must:
- 썸네일 이미지 노출. 실패 또는 없음이면 fallback image 사용.
- 작품명 노출. 길이 초과 시 말줄임 처리.
- 평점 노출. 예: `★4.9`, 평점 없으면 `-/5`.
- 우하단에 상태 설정 버튼 노출.
- 상태 변경은 사용자 계정에 저장되고, 모든 화면에서 동기화되어야 한다.

상태 설정 버튼:
| 상태 코드 | 표기명 | 아이콘 | 의미 |
| --- | --- | --- | --- |
| KEEP | 볼 예정 | 📌 | 볼 예정 |
| WATCHING | 보는 중 | ▶️ | 감상 진행 중 |
| DONE | 완료 | ✅ | 감상 완료 |
| STOPPED | 중단 | ⛔ | 감상 중단/보류 |
| CANCEL | 취소 | ✖️ | 상태 해제 |

상호작용:
- 기본 상태는 `+` 아이콘.
- 클릭 시 드롭다운 표시: `볼예정`, `보는중`, `완료`, `중단`, `취소`.
- 선택 시 아이콘이 상태별 아이콘으로 변경된다.
- 선택된 항목에는 체크 표시를 보여준다.
- `취소` 선택 시 상태를 해제하고 선택 표시를 제거한다.
- 비회원이 클릭하면 `로그인이 필요한 기능입니다` 팝업 후 로그인/회원가입으로 이동한다.

### 4-2. 무한 스크롤 `CMP-SCROLL-001`
적용 범위: 리스트/그리드형 작품 노출 화면. 가로 캐러셀은 예외.

Must:
- 방향은 세로만.
- 최초 로드는 30개. 기준: 가로 6 × 세로 5.
- 추가 로드도 30개 단위.
- 하단 임계 지점 진입 시 추가 로드.
- 로딩 중에는 선형 인디케이터 또는 스피너 노출.
- 데이터가 더 없으면 로드 중단. 종료 문구는 표시하지 않는다.
- 0건이면 빈 상태를 노출한다.

Nice:
- 로드 실패 시 자동 재시도 3회.

### 4-3. 작품 평가 `CMP-EVAL-001`
적용 위치: 작품 상세, 홈 인기 평가.

변형:
- 상세 페이지: 썸네일 없음.
- 홈 인기 평가: 썸네일 있음.

Must:
- 별점 1~5점 입력 필수.
- 평가글은 10자 이상 1,000자 이하.
- 작성자 닉네임 노출.
- 비회원의 작성/공감 클릭은 로그인 유도.

Nice:
- 공감 투표와 카운트 반영. 1인 1회.
- 작품당 1회만 평가 작성.
- 작성 후 입력창 비활성화.
- 정렬은 최신순 고정.

### 4-4. 픽카드 `CMP-PCARD`
올블픽 추천 카드. 홈과 웹툰/애니 화면에서 사용.

Must:
- 왼쪽 기준작, 오른쪽 추천작 구조.
- 각 카드 구성: 썸네일, 작품명, 좋아요 수, 최초 추천자 닉네임.
- 썸네일 클릭 시 작품 상세 이동.
- 기준작 하단 텍스트: `좋아하신다면`.
- 추천작 하단 텍스트: `추천해요`.

### 4-5. 푸터 `CMP-FOOTER-001`
전 화면 하단 공통.

Must:
- 문의 안내: `xhdsxhd@gmail.com 연락처로 알려주세요.`
- 비상업 MVP 검증 목적 안내.
- 업체명: `팀 2게진짜_최종 제공`.
- AniList API, TMDB 출처 고지. 약관상 필수라 삭제 금지.
- 각 API 출처명은 원문 링크 연결.
- 전 화면 하단에 노출.

## 5. 화면별 기능명세

### 5-1. 메인 홈 `SCR-HOME-001`
서비스 진입 최초 화면. 큐레이션과 통합 검색 진입점.

공통:
- 접근 권한은 전체 공개.
- 섹션 노출은 웹/모바일 동일.
- 인기작 순위는 기본 6개, 최대 20개.

GNB Must:
- 로고 클릭 시 홈 이동.
- 홈 메뉴 클릭 시 홈 이동, 현재 화면이면 active.
- 웹툰/애니 탭 이동.
- 올블픽 이동.

검색/계정 Must:
- 검색창은 키워드 입력 시 드롭다운 리스트업.
- 공백 또는 미입력 검색 실행 방지.
- 검색 범위: 전체/애니/웹툰.
- 작품명만 검색 가능.
- 회원 사용자 아이콘 클릭 시 마이페이지 이동.
- 비회원은 로그인/회원가입 링크 노출.
- 로그인 성공 시 홈으로 이동.

홈 섹션:
- 히어로 배너: 서비스 소개 배너. Nice.
- 인기작 순위 TOP 20: 애니/웹툰 전환, 순위 카드, 좌우 이동, 상세 이동. Nice.
- 인기 평가글 카드 3개: 썸네일, 작품명, 평가글, 작성자, 공감수. MVP는 최신순. Nice.
- 최근 추가된 애니 추천: 3개 카드 가로 노출, 좌우 캐러셀. Must.
- 최근 추가된 웹툰 추천: 애니와 동일. Must.

### 5-2. 로그인/회원가입 `SCR-AUTH-001`
비회원의 회원 전용 기능 클릭 시 진입.

MVP 범위:
- 아이디와 닉네임 고유값 검증.
- 중복 방지.
- 소셜 로그인, 이메일 인증 제외.
- DB 설계: `아이디(key)`, `비밀번호(value)`.

로그인 Must:
- 아이디 입력. 미입력 차단.
- 비밀번호 입력. 마스킹 및 미입력 차단.
- 로그인 버튼 클릭 시 인증, 성공하면 홈 이동.
- 인증 실패 시 안내 문구와 필드 하단 에러.
- 회원가입 페이지 이동 링크.

회원가입 Must:
- 닉네임 고유값 검증 및 중복 방지.
- 아이디 고유값 검증 및 중복 방지.
- 비밀번호 마스킹, 20자 이하 제한.
- 필드 하단 유효성 안내:
  - `중복된 ID 입니다`
  - `사용 가능한 ID 입니다`
  - `비밀번호는 20자 이하로 작성해주세요.`
- 계정 생성 후 회원가입 완료 화면으로 이동.
- 로그인 페이지 이동 링크.

회원가입 완료 Must:
- `회원가입 완료! 로그인 해주세요` 문구.
- 로그인 버튼 클릭 시 로그인 페이지 이동.
- 자동 로그인은 하지 않는다.

### 5-3. 애니 탭 `SCR-ANIME-001`
GNB, 검색, 푸터는 홈과 동일. 썸네일은 `CMP-THUMB`.

세컨더리 탭:
- 올블루추천
- 전체

올블루추천 Nice:
- 인기순위 TOP20. 실시간/주간/월간/연간 캐러셀.
- 완결된 애니 일부 노출 후 전체보기.
- 2026 SS 신작 애니 캐러셀.
- 최근 추가된 작품 캐러셀.
- MVP에서는 실제 순위 대신 임의 작품 사용 가능.

전체 탭:
- 장르 필터: 전체/액션/판타지/로맨스/드라마/이세계. Nice.
- 작품 그리드: 연재/완결 뱃지, 썸네일, 제목. Must.
- 최초 30개, 이후 30개씩 세로 무한 스크롤. Must.

### 5-4. 웹툰 탭 `SCR-WEBTOON-001`
GNB, 검색, 푸터는 홈과 동일. 썸네일은 `CMP-THUMB`.

세컨더리 탭:
- 연재중
- 완결

연재중 Nice:
- 플랫폼별 섹션. MVP는 네이버/카카오.
- 섹션당 가로 8 × 세로 3 최초 표시.
- 플랫폼별 가로 캐러셀.
- 비정기 연재 요일 단위.

완결:
- 장르 필터. Nice.
- 작품 그리드 가로 6 × 세로 5. Must.
- 첫 30개 후 30개씩 세로 무한 스크롤. Must.
- 다음 작품 선로딩. Nice.

### 5-5. 작품 상세 `SCR-DETAIL-001`
개별 작품의 정보, 감상 상태, 추천, 평가 통합 화면.

기본 정보 Must:
- 작품명.
- 대표 썸네일. 실패 시 fallback.
- 평균 평점과 참여 수. 평점 없으면 `-/5`.
- 장르 리스트.

액션 버튼 Must:
- 시청 상태 버튼: 볼예정/보는중/완료/중단.
- 같은 작품에 하나의 상태만 활성.
- 다른 상태 클릭 시 이전 상태 비활성화 후 현재 상태 활성화.
- 비회원 클릭 시 로그인 유도.
- 상태 데이터는 썸네일 상태와 동일 소스를 사용한다.

개요/플랫폼 Must:
- 작품 개요. 길이 초과 시 더보기 접힘.
- 감상 가능 플랫폼 노출.
- OTT 버튼 클릭 시 해당 시청 화면 링크 이동.

메타데이터 Must:
- 공통: 작품명, 장르, 개요, 감상 플랫폼, 썸네일, 타입, 태그.
- 애니: 원작, 원작사, 감독, 화수, 방영기간, 방영상태.
- 웹툰: 원작, 글작가, 그림작가, 화수, 연재기간, 연재상태.

상세 내 올블픽 Must:
- `이 작품 좋아하면 이것도` 추천 리스트 6개.
- 없으면 `아직 추천한 작품이 없어요`.
- 전체 보기 클릭 시 유저 추천 목록 또는 올블픽 상세로 이동.

상세 내 올블픽 Nice:
- 유저가 추천 등록.
- 추천 애니/웹툰 선택 버튼.
- 내 추천 작성하기.

시리즈 순서 Nice:
- 연관 작품 트리.
- 방영 순 기본.
- 세계관 시간 순 선택 가능.

작품 평가 섹션 Must:
- 작성자, 별점 1~5, 평가글 10~1,000자, 등록 버튼.
- 글자수 위반 시 alert:
  - `글자수는 10자에서 1000자 사이로 입력해주세요.`
- 등록된 평가 UI: 작성자, 별점, 내용, 공감수, 작성일.
- 평가글 클릭 시 평가 상세 페이지 이동.
- 비회원 작성/공감은 로그인 유도.
- 평가 목록은 최근 등록 순, 가로 3 × 세로 제한 없음.

작품 평가 섹션 Nice:
- 작품당 1회만 작성.
- 공감 1인 1회.

### 5-6. 작품 평가 상세 `SCR-EVAL-001`
홈 인기 평가 카드 또는 작품 상세 평가 클릭 시 페이지 전환으로 진입.

Must:
- 작성자 닉네임과 별점 헤더.
- 평가글 10~1,000자.
- 등록 버튼.
- 페이지 전환 방식. 플로팅/모달 아님.
- 작성자 본인은 수정/삭제 가능.
- 수정은 수정 페이지 또는 수정 상태로 이동 후 작성/취소 버튼 제공.
- 삭제는 확인 모달 없이 즉시 삭제.
- 취소 시 기존 평가 상세로 이동.

Nice:
- 타 유저는 공감/팔로우 버튼.
- 공감 투표.
- 팔로우.

### 5-7. 올블픽 `SCR-OLLPICK-001`
GNB 올블픽 또는 작품 상세 유저추천 섹션에서 진입.

핵심 데이터 단위:
- 추천 1건 = 기준작 A + 추천작 B + 이유글들.
- 정렬은 최신순.

상단 Must:
- 최신 반영 추천 카드 가로 노출.
- 카드 클릭 시 추천 상세 이동.

내가 본 작품 영역 Must:
- 본 작품 있음: 추천 카드 리스트 + 사이드 패널.
- 본 작품 없음: 빈 상태 + `본 작품 등록하러 가기` CTA.
- 본 작품 없음이면 동의 버튼 비활성.
- 본 작품 기준은 `보는중`, `완료`.

추천 카드 Must:
- 기준작 A 클릭: 해당 작품 추천 페이지로 이동.
- 추천작 B 클릭: 추천작 기준 추천 섹션으로 이동.
- 이유 더보기.
- 동의 버튼.

사이드 패널:
- 내 추천 작성하기. 회원 전용. Must.
- 나의 추천 활동. Nice.
- 아직 추천 없는 내 작품. Nice.

투고 로직 Must:
- 기준작 A에 대해 연관 추천작 B 투고.
- A와 B 모두 본인이 `보는중` 또는 `완료` 상태여야 투고 가능.
- 이유는 최신순 누적.
- 같은 A/B 재투고 시 새 카드 생성하지 않고 기존 카드 동의 +1, 이유 추가.
- 동의 시 인정 이유 필수 입력.

투고문 작성 Must:
- 메인 작품 검색/선택. 보는중/완료만 검색.
- 추천할 작품 검색/선택. 보는중/완료만 검색.
- 추천 이유 10자 이상 200자 이하.
- 저장/투고.
- 취소.

투고문 상세 Must:
- 기준작 클릭 시 작품 상세 이동.
- 추천작 클릭 시 작품 상세 이동.
- 누적 이유글 최신순 노출.
- 동의 +1 및 인정 이유 등록.

동의 처리 Must:
- 동의 클릭 시 해당 추천 동의 수 +1.
- 인정 이유 등록 후 이유란 누적.
- 본 작품 없으면 비활성.
- 동일 추천에 1인 1회.

삭제/수정 Must:
- 올블픽 투고문은 삭제만 구현.
- 삭제 경로는 마이페이지 내에서만 제공.

### 5-8. 유저추천 상세 `SCR-USERREC-001`
GNB 올블픽 또는 작품 상세 `모든 추천 보기`에서 진입.

Must:

- 이유글은 닉네임 + 이유글 리스트.
- 이유 더보기.
- 추천 작품 클릭 시 작품 상세 이동.

Nice:
- 이유글 최신순 정렬.

개요 탭:

## 7. MVP 구현 우선순위 제안

### 7-1. 1차 MVP Must
1. Next.js 프로젝트 기본 구조.
2. 공통 GNB, 검색 UI, 푸터.
3. 로그인/회원가입 localStorage 기반 MVP.
4. 작품 mock DB.
5. 작품 썸네일 카드와 상태 드롭다운.
6. 홈 최근 추가 애니/웹툰 추천 캐러셀.
7. 애니 전체 탭 30개 그리드 + 추가 로드.
8. 웹툰 완결 탭 30개 그리드 + 추가 로드.
9. 작품 상세 기본 정보, 상태 버튼, 개요, 플랫폼, 메타데이터.
10. 작품 평가 작성 및 목록.
11. 마이페이지 보관함, 내가 쓴 평가 글.
12. 올블픽 기본 목록, 작성, 동의, 이유 누적.

### 7-2. 후순위 Nice
- 인기작 실제 순위.
- 홈 인기 평가 공감순.
- 공감/팔로우 고도화.
- 작품당 평가 1회 제한.
- 시리즈 트리.
- 플랫폼별 웹툰 연재중 캐러셀 고도화.
- 성능 최적화 선로딩.

## 8. 결정 필요 사항
아래는 구현 전 또는 구현 중 명확히 정해야 하는 항목이다.

1. `CANCEL` 상태가 단순 상태 해제인지, 별도 기록으로 남기는지.
2. 작품 데이터 소스: mock 자체 DB, AniList/TMDB 연동, 웹툰 별도 CSV 중 무엇을 MVP 기준으로 할지.
3. 검색 결과 화면의 라우팅과 UI 범위.
4. 올블픽 중복 투고 시 `동의 +1`을 본인 재투고에도 허용할지.
5. 평가 수정/삭제의 1차 MVP 포함 여부.
6. 회원 인증을 localStorage로 시작할지, Supabase 등 실제 DB로 바로 갈지.
